Lesson 0001 · Go + D&D 5e

The d20 Check

Writing D&D 5e's core resolution mechanic in Go — the first piece of your DM bot's rules engine.

Everything in D&D resolves through one atomic operation: roll a d20, add a modifier, compare to a target number. If you build exactly this in Go, you have the foundation every other rule compiles against. That's this lesson.

D&D 5e The Ability Check

You've done this thousands of times in BG3 without reading the rule. Here it is written out.[SRD 5.2]

d20 + ability modifier (+ proficiency bonus, if applicable) ≥ DC  →  Success

The Difficulty Class (DC) is a fixed number set by the DM — 10 for easy tasks, 15 for medium, 20 for hard, 25 for very hard. The ability modifier comes from one of the six core scores:

ScoreModifierScoreModifier
1−512–13+1
2–3−414–15+2
4–5−316–17+3
6–7−218–19+4
8–9−120+5
10–11+0

The formula behind the table:

modifier = ⌊(score − 10) / 2⌋

Every two points of score = one point of modifier. The floor matters — odd scores below 10 round down into negative territory. Score 9 gives −1, not 0. This will matter when we write the Go code.

Go Writing the Resolver

Your Python instincts are almost right. The differences that will trip you up are small but rigid — Go enforces them at compile time, not at runtime.

Packages and Imports

Every Go file declares its package. The package name matches its directory. This file will live at pkg/dice/dice.go, so it belongs to package dice.[Tour of Go]

Python
import random
import math
Go
import (
    "fmt"
    "math"
    "math/rand"
)

One hard rule: the Go compiler refuses to compile if you import a package you don't use. This is not a linter. It is the language. Unused imports are a compile error.

Functions and Exported Names

Types come after names, not before. The return type is declared after the parameter list.

Python
def roll(sides: int) -> int:
    return random.randint(1, sides)
Go
func Roll(sides int) int {
    return rand.Intn(sides) + 1
}

The capital R in Roll is not style. A capitalized identifier is exported from the package — accessible to callers. Lowercase is package-private. This replaces Python's __dunder and single-underscore conventions with a single, consistent rule.

Gotcha Floor Division vs Truncation

Python's // is floor division — it rounds toward negative infinity. Go's / on integers truncates toward zero. They differ on negative numbers:

Python (floor)
-1 // 2  →  -1  ✓
Go (truncation)
-1 / 2   →   0  ✗

Score 9: (9−10)/2 = −1/2. Python gives −1 (correct for D&D). Go gives 0 (wrong). Fix it with math.Floor, which also introduces Go's explicit type conversion — there's no implicit numeric coercion:

func AbilityModifier(score int) int {
    return int(math.Floor(float64(score-10) / 2))
}

float64(score-10) converts the integer to a float. int(...) converts back. Go makes every conversion visible in the code — types never silently widen or narrow.

Errors Are Values

Go has no exceptions. Errors are return values. A function that can fail returns (result, error) — two values at once:

Python (raises)
def roll(sides):
    if sides < 2:
        raise ValueError("need ≥2 sides")
    return random.randint(1, sides)
Go (returns error)
func Roll(sides int) (int, error) {
    if sides < 2 {
        return 0, fmt.Errorf("need ≥2 sides, got %d", sides)
    }
    return rand.Intn(sides) + 1, nil
}

The caller receives both values and decides what to do with the error. It cannot accidentally ignore it — if the caller assigns both values, the compiler enforces that both are used (or explicitly discarded with _).

Structs Replace Classes

Go has no classes. A struct is a named grouping of typed fields. Methods can be attached separately. For returning a rich result from AbilityCheck, a struct is exactly right:

type CheckResult struct {
    Roll     int
    Modifier int
    Total    int
    DC       int
    Success  bool
}

CheckResult{} creates a zero-value struct — every field set to its zero: int → 0, bool → false, string → "". Go doesn't have None; it has zero values. This matters for error returns.

The Complete File

This is pkg/dice/dice.go — the rules engine's foundation:

package dice

import (
	"fmt"
	"math"
	"math/rand"
)

// Roll returns a random number from 1 to sides, inclusive.
func Roll(sides int) (int, error) {
	if sides < 2 {
		return 0, fmt.Errorf("Roll: need at least 2 sides, got %d", sides)
	}
	return rand.Intn(sides) + 1, nil
}

// AbilityModifier converts a D&D 5e ability score to its modifier.
// Uses floor division — Go's integer division truncates toward zero,
// which gives the wrong result for odd scores below 10.
func AbilityModifier(score int) int {
	return int(math.Floor(float64(score-10) / 2))
}

// CheckResult holds the outcome of a D&D 5e ability check.
type CheckResult struct {
	Roll     int
	Modifier int
	Total    int
	DC       int
	Success  bool
}

// AbilityCheck resolves a D&D 5e ability check against a Difficulty Class.
func AbilityCheck(abilityScore, dc int) (CheckResult, error) {
	roll, err := Roll(20)
	if err != nil {
		return CheckResult{}, err
	}
	mod := AbilityModifier(abilityScore)
	total := roll + mod
	return CheckResult{
		Roll:     roll,
		Modifier: mod,
		Total:    total,
		DC:       dc,
		Success:  total >= dc,
	}, nil
}
Go idiom: := vs =
roll, err := Roll(20) — the := operator declares and assigns in one step. Use it for new variables. Plain = assigns to an existing variable. You'll see := constantly in Go; it's how most local variables are born.
