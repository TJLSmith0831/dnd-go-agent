Lesson 0010 · D&D 5e

Skills & Proficiency

The modifier system that separates a trained rogue from a blundering paladin — and the last rules layer the Referee agent needs to adjudicate any check.

You can already resolve a raw ability check: d20 + modifier vs DC. But most checks in play aren't raw — they're skill checks, and the difference is proficiency. A ranger tracking footprints through a forest rolls Wisdom, but they also add their proficiency bonus if they're trained in Survival. That bonus is the gap between "roll and hope" and "reliably competent." The Referee agent needs to know both numbers to resolve anything a player attempts.

D&D 5e The 18 Skills

Every skill is anchored to one of the six ability scores. When you call for a skill check, you're calling for a specific ability score applied to a specific context.

AbilitySkills
StrengthAthletics
DexterityAcrobatics, Sleight of Hand, Stealth
IntelligenceArcana, History, Investigation, Nature, Religion
WisdomAnimal Handling, Insight, Medicine, Perception, Survival
CharismaDeception, Intimidation, Performance, Persuasion
Constitution(no skills — used for concentration saves and HP)

When a player says "I try to pick the lock," the Referee maps that to Dexterity (Thieves' Tools) — technically a tool proficiency, not a skill, but resolved identically. "I try to recall the dragon's weaknesses" → Intelligence (Arcana). "I try to read the merchant's expression" → Wisdom (Insight).

D&D 5e Proficiency Bonus

Every character has a proficiency bonus determined solely by their character level — not class, not ability scores.

Character LevelProficiency Bonus
1–4+2
5–8+3
9–12+4
13–16+5
17–20+6

This bonus is added to attack rolls, saving throws, and skill checks — but only when the character is proficient. Proficiency is a binary: you either have it or you don't. A wizard proficient in Arcana adds the bonus; a fighter who isn't doesn't.

Skill check formula

proficient: d20 + ability modifier + proficiency bonus ≥ DC not proficient: d20 + ability modifier ≥ DC

Expertise

Rogues and Bards can gain expertise in certain skills: they add double their proficiency bonus. A level-5 Bard (proficiency +3) with expertise in Persuasion adds +6 to every Charisma (Persuasion) check. This is the only multiplier in 5e — advantage/disadvantage applies to the die, not the modifier.

expertise: d20 + ability modifier + (proficiency bonus × 2) ≥ DC

D&D 5e Passive Checks

Sometimes the DM needs to know if a character notices something without them actively looking — sneaking past a guard, spotting a hidden door while walking past. Rather than calling for a roll, the DM uses the character's passive check score: a fixed number representing their baseline competence.

passive check = 10 + ability modifier (+ proficiency bonus if proficient)

The most common is Passive Perception. A character with WIS 14 (+2) and proficiency in Perception (+3) has a passive Perception of 15. Any hidden creature or object with a Stealth/DC below 15 is spotted automatically, without a roll.

BG3 note: that number next to the eye icon on each character's portrait — that's their passive Perception. When you walk past a hidden trap and a character automatically spots it, their passive Perception beat the trap's DC.

Bot The Referee Agent's Decision Tree

When a player sends a free-text action, the Referee agent must work through four questions before rolling:

  1. Is a check even needed? Trivial tasks (walking through an open door) auto-succeed. Impossible tasks (jumping to the moon) auto-fail. Only uncertain outcomes need a roll.
  2. Which skill applies? Map the action to one of the 18 skills (or a raw ability check if no skill fits).
  3. Is the character proficient? Look up their character sheet. Add proficiency bonus if yes, expertise if applicable.
  4. What's the DC? Easy 10, Medium 15, Hard 20, Very Hard 25, Nearly Impossible 30.

Then call SkillCheck and narrate the result.

func SkillCheck(score, profBonus int, proficient bool, dc int) (CheckResult, error) {
    roll, err := Roll(20)
    if err != nil {
        return CheckResult{}, err
    }
    mod := AbilityModifier(score)
    if proficient {
        mod += profBonus
    }
    total := roll + mod
    return CheckResult{Roll: roll, Modifier: mod, Total: total, DC: dc, Success: total >= dc}, nil
}

func PassiveCheck(score, profBonus int, proficient bool) int {
    mod := AbilityModifier(score)
    if proficient {
        mod += profBonus
    }
    return 10 + mod
}

For expertise, pass profBonus * 2 — no new function needed. The caller doubles it before passing it in.
