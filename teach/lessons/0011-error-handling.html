Error Handling

How Go distinguishes error identity from error context — and why the Referee agent needs both.

You've been writing if err != nil { return err } since Lesson 1. That's the right instinct — errors are values, not exceptions. But as the bot grows, returning a raw error isn't enough. The Referee agent needs to know why something failed, not just that it failed. "No skill matched the player's input" is recoverable — ask for clarification. "Dice roll failed" is a bug — log it and continue. Go's error wrapping tools let you carry both context and identity through the call stack.

Go Errors Are an Interface

The error type is just an interface:

type error interface {
    Error() string
}

Any type with an Error() string method satisfies it. errors.New("something went wrong") returns a pointer to a small unexported struct that does exactly that. You've been using this the whole time without the machinery mattering — now it does.

Go Sentinel Errors

A sentinel error is a package-level variable that callers can compare against. It expresses a kind of failure, not the details of a specific failure:

var ErrUnknownSkill = errors.New("unknown skill")

The caller checks identity with errors.Is:

if errors.Is(err, referee.ErrUnknownSkill) {
    // ask the player to clarify
}
Python analogy: sentinel errors are like specific exception classes (except KeyError). errors.Is is the equivalent of isinstance(err, KeyError) — but it works through the wrapping chain.

Go Wrapping Errors with %w

When a low-level error bubbles up through several layers, each layer should add context before passing it along. fmt.Errorf with the %w verb does this:

// dice.go — low-level error
return fmt.Errorf("Roll: need at least 2 sides, got %d", sides)

// referee.go — adds context, preserves the original
return fmt.Errorf("resolving %q for %s: %w", input, character, err)

The result is an error chain: the referee error wraps the dice error. %w is what creates the chain — %s would just convert the error to a string and sever it.

VerbEffectChain preserved?
%wwraps the errorYes — errors.Is and errors.As traverse it
%s or %vconverts to stringNo — identity is lost

Go errors.Is and errors.As

errors.Is — check identity through the chain

// true even if err is wrapped several layers deep
errors.Is(err, ErrUnknownSkill)

This unwraps the chain until it finds a match or runs out of links. Direct comparison (err == ErrUnknownSkill) only checks the outermost error — it misses wrapped errors.

errors.As — extract a typed error from the chain

var refErr *RefereeError
if errors.As(err, &refErr) {
    // refErr is now populated — access its fields
    log.Printf("bad input: %q", refErr.Input)
}

errors.As is for typed errors — when you need more than just "did this kind of error occur?" and want to read the error's fields. It unwraps the chain looking for anything assignable to the target type.

Go Custom Error Types

When a sentinel isn't enough — when the caller needs context like which input caused the failure — define a struct that implements error:

type RefereeError struct {
    Input  string // what the player typed
    Reason string // why resolution failed
}

func (e *RefereeError) Error() string {
    return fmt.Sprintf("referee: %q: %s", e.Input, e.Reason)
}

Return it like any other error. The caller uses errors.As to get it back out with its fields intact.

The two failure modes the Referee must distinguish:
ErrUnknownSkill — player said something the Referee couldn't map to a skill. Recoverable: reply "Could you be more specific?"
Any other error — a bug in dice rolling or the LLM call. Not recoverable mid-session: log it, skip the turn, keep running.

Go TDD: Build pkg/referee

Create pkg/referee/referee.go and pkg/referee/referee_test.go. The Referee maps a player's free-text input to a skill. If it can't, it returns a *RefereeError wrapping ErrUnknownSkill.

Red — referee_test.go
package referee

import (
    "errors"
    "testing"
)

func TestResolveSkill(t *testing.T) {
    t.Run("recognises stealth", func(t *testing.T) {
        skill, err := ResolveSkill("I try to sneak past the guard")
        if err != nil {
            t.Fatalf("unexpected error: %v", err)
        }
        if skill != SkillStealth {
            t.Errorf("got %v, want Stealth", skill)
        }
    })

    t.Run("returns ErrUnknownSkill for unrecognised input", func(t *testing.T) {
        _, err := ResolveSkill("I want to do a cool thing")
        if !errors.Is(err, ErrUnknownSkill) {
            t.Errorf("expected ErrUnknownSkill, got %v", err)
        }
    })

    t.Run("RefereeError carries the original input", func(t *testing.T) {
        _, err := ResolveSkill("I want to do a cool thing")
        var refErr *RefereeError
        if !errors.As(err, &refErr) {
            t.Fatalf("expected *RefereeError in chain, got %T", err)
        }
        if refErr.Input != "I want to do a cool thing" {
            t.Errorf("Input: got %q", refErr.Input)
        }
    })
}
Green — referee.go
package referee

import (
    "errors"
    "fmt"
    "strings"
)

type Skill string

const (
    SkillStealth     Skill = "stealth"
    SkillPerception  Skill = "perception"
    SkillPersuasion  Skill = "persuasion"
    SkillAthletics   Skill = "athletics"
    SkillArcana      Skill = "arcana"
)

var ErrUnknownSkill = errors.New("unknown skill")

type RefereeError struct {
    Input  string
    Reason string
}

func (e *RefereeError) Error() string {
    return fmt.Sprintf("referee: %q: %s", e.Input, e.Reason)
}

func (e *RefereeError) Unwrap() error {
    return ErrUnknownSkill
}

// keywords maps input fragments to skills.
var keywords = map[string]Skill{
    "sneak":    SkillStealth,
    "hide":     SkillStealth,
    "notice":   SkillPerception,
    "spot":     SkillPerception,
    "convince": SkillPersuasion,
    "persuade": SkillPersuasion,
    "climb":    SkillAthletics,
    "jump":     SkillAthletics,
    "recall":   SkillArcana,
    "magic":    SkillArcana,
}

func ResolveSkill(input string) (Skill, error) {
    lower := strings.ToLower(input)
    for kw, skill := range keywords {
        if strings.Contains(lower, kw) {
            return skill, nil
        }
    }
    return "", &RefereeError{Input: input, Reason: "no keyword matched"}
}

The key line is Unwrap() error on RefereeError — that's what makes errors.Is(err, ErrUnknownSkill) work when err is a *RefereeError. Without Unwrap, errors.Is can't see through to the sentinel.

Note: keyword matching is a placeholder. In the real bot, ResolveSkill will pass the player's input to the LLM and ask it to identify the skill — far more robust than string matching. The error types and the interface stay the same; only the matching logic changes.
